# 개발자-후원자 매칭 플랫폼 설계

> 작성일: 2026-01-29
> 수정일: 2026-01-29 (법적 검토 반영)

## 1. 프로젝트 개요

AI 발달로 증가하는 1인 개발자(창업자)와 후원자를 연결하는 플랫폼.

### 핵심 특징
- **관계**: 개발자 1명 : 후원자 N명
- **플랫폼 역할**: 매칭/중개만 (계약은 당사자 간 개인 계약)
- **품질 관리**: 관리자가 프로젝트 심사 + 주간 보고 의무화
- **자금 흐름**: PG사 에스크로 활용 (플랫폼이 돈을 만지지 않음)
- **수익 모델**: 후원 성사 시 중개 수수료 (7~10%)

---

## 2. 액터

| 액터 | 설명 |
|-----|-----|
| 개발자 | 1인 창업자, 프로젝트 등록 및 후원 유치 |
| 후원자 | 프로젝트에 후원하는 사람 (서포터) |
| 관리자 | 플랫폼 운영자, 프로젝트 심사 및 품질 관리 |

---

## 3. 유스케이스

### 3.1 개발자

**인증/가입**
- 회원가입 (OAuth)
- 로그인/로그아웃

**프로젝트 등록**
- 프로젝트 계획서 작성 (목표금액, 최소금액, 개발기간, 기술스택, 리워드 등)
- 계획서 제출 → 심사 대기

**프로젝트 관리**
- 심사 결과 확인 (승인/반려)
- 반려 시 계획서 수정 후 재제출
- 주간 진행상황 보고서 제출
- 후원 모집 현황 조회 (현재 금액, 참여 후원자 수)

**후원 진행**
- 목표액 달성 알림 수신
- 유튜브 라이브 진행 (후원자들과 소통)
- MVP 제출
- MVP 검토 결과 확인 (통과/실패)
- 최종 후원금액 확인 후 진행 여부 결정 (최소금액 미달 시)
- 정산금 수령 (본 개발 시작 후, 마일스톤별)

**종료**
- 모집 종료 알림 수신 (미달성 시)
- 프로젝트 완료 처리

### 3.2 후원자

**인증/가입**
- 회원가입 (OAuth)
- 로그인/로그아웃

**프로젝트 탐색**
- 프로젝트 리스트 조회
- 프로젝트 검색/필터 (분야, 기술스택, 목표금액 등)
- 추천 프로젝트 조회
- 프로젝트 상세 조회 (계획서, 진행상황 보고서)

**후원**
- 프로젝트 후원 (PG 결제 → 에스크로 예치)
- 후원 금액 추가
- 후원 취소 (모집 기간 중)
- 내 후원 목록 조회

**후원 진행**
- 목표액 달성 알림 수신
- 유튜브 라이브 참여
- MVP 결과물 확인
- 계속 후원 / 철회 결정
- 주간 보고서 열람 (본 개발 시작 후)

**종료**
- 모집 종료 알림 수신 (미달성 시)
- 환불 처리 (에스크로에서 자동 반환)
- 리워드 수령 (프로젝트 완료 시)

### 3.3 관리자

**프로젝트 심사**
- 심사 대기 프로젝트 목록 조회
- 계획서 검토 (실현가능성, 법적 제약)
- 프로젝트 승인/반려 (반려 시 사유 작성)

**품질 관리**
- 주간 보고서 미제출 개발자 확인
- 경고/제재 조치 (보고서 미제출 시)
- 프로젝트 강제 종료 (문제 발생 시)

**MVP 검증**
- MVP 제출물 검토
- MVP 통과/실패 판정

**정산 관리**
- 마일스톤 달성 확인
- PG사에 정산 승인 API 호출

**회원 관리**
- 개발자/후원자 목록 조회
- 문제 회원 제재/탈퇴 처리

## 4. 후원 프로세스 흐름

```
1. [프로젝트 등록] 개발자가 계획서 작성 (목표금액 + 최소금액 + 개발기간 설정)
        ↓
2. [심사] 관리자가 계획서 검토 → 승인/반려
        ↓
3. [모집] 후원자들이 PG 결제 (에스크로 예치)
        ↓
   ┌────┴────┐
   ↓         ↓
 목표액     기간만료
 달성       (미달성)
   ↓         ↓
   │       [종료] 에스크로에서 전액 환불
   ↓
4. [유튜브 라이브] 개발자-후원자 소통
        ↓
5. [MVP 개발] 1개월, 무급
        ↓
6. [MVP 검증] 관리자가 결과물 검토
        ↓
   ┌────┴────┐
   ↓         ↓
 통과       실패
   ↓         ↓
   │       [종료] 에스크로에서 환불
   ↓
7. [후원자 결정] 계속 후원 or 철회
        ↓
8. [최종금액 확정]
        ↓
   ┌────┴────┐
   ↓         ↓
 ≥최소금액  <최소금액
   ↓         ↓
 자동진행   [개발자 선택]
   ↓         ↓
   │    ┌────┴────┐
   │    ↓         ↓
   │  진행함    포기함
   │    ↓         ↓
   └────┴────→    │
        ↓         ↓
9. [본 개발 진행]  [종료] 환불
   주간보고 + 마일스톤별 정산
        ↓
10. [완료] 리워드 제공
```

## 5. 프로젝트 상태

| 상태 | 설명 |
|-----|-----|
| `DRAFT` | 계획서 작성 중 |
| `PENDING_REVIEW` | 심사 대기 |
| `UNDER_REVIEW` | 심사 중 |
| `REJECTED` | 반려 (수정 후 재제출 가능) |
| `FUNDING` | 모집 중 |
| `MVP_IN_PROGRESS` | MVP 개발 중 (1개월) |
| `MVP_REVIEW` | MVP 검증 중 |
| `SUPPORTER_DECISION` | 후원자 결정 대기 |
| `DEVELOPER_DECISION` | 개발자 진행 여부 결정 대기 |
| `ACTIVE` | 본 개발 진행 중 |
| `COMPLETED` | 완료 |
| `CLOSED` | 종료 (사유는 별도 필드) |

**종료 사유 (CLOSED일 때):**
- `FUNDING_FAILED` - 모집 실패
- `MVP_FAILED` - MVP 검증 실패
- `DEVELOPER_QUIT` - 개발자 포기
- `DEVELOPER_REJECTED` - 개발자가 최소금액 미달로 진행 거부
- `ADMIN_CLOSED` - 관리자 강제 종료

## 6. 자금 흐름 (PG 에스크로)

### 6.1 구조

```
[후원자]                    [PG사 에스크로]              [개발자]
   │                             │                        │
   │──(카드/계좌결제)──────────→ │                        │
   │                        예치 보관                     │
   │                             │                        │
   │                    (마일스톤 달성)                   │
   │                             │                        │
   │                      플랫폼이                        │
   │                   "정산 승인" API 호출               │
   │                             │                        │
   │                             │───(정산금)───────────→ │
   │                             │                        │
   │←─────(환불)─────────────────│   (모집실패/MVP실패 시) │
```

### 6.2 핵심 원칙

**플랫폼은 돈을 만지지 않는다**
- 모든 결제는 PG사로 직접 입금
- 플랫폼은 정산 "신호"만 보냄
- 환불도 PG사 에스크로에서 직접 처리

### 6.3 정산 규칙

**마일스톤 기반 정산:**
- 개발 기간을 N개 마일스톤으로 분할
- 각 마일스톤 달성 + 주간보고 제출 시 해당 회차 정산
- 관리자가 확인 후 PG사에 정산 승인 API 호출

**환불 규칙:**
- 모집 기간 중: 전액 환불 가능
- MVP 실패 시: 전액 환불
- 후원자 철회 시: 미정산분 환불
- 개발자 중도 포기 시: 미정산분 환불

### 6.4 플랫폼 수익

```
후원금 100만원 기준
├─ 개발자 정산: 90만원 (90%)
├─ PG 수수료: 약 3만원 (3%)
└─ 플랫폼 수수료: 7만원 (7%)
```

## 7. 리워드 구조

후원자에게 금전적 수익이 아닌 **비금전적 보상** 제공:

| 리워드 예시 | 설명 |
|-----------|-----|
| 베타테스트 참여권 | 출시 전 미리 사용 |
| 엔딩 크레딧 기재 | 후원자 이름 표시 |
| 한정판 디지털 굿즈 | 배경화면, 아이콘 등 |
| 1:1 피드백 세션 | 개발자와 직접 소통 |
| 얼리버드 이용권 | 출시 후 무료/할인 이용 |

**주의**: 수익 배분(Revenue Share)은 증권법 위반이므로 절대 불가
