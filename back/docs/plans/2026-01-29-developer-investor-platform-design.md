# 개발자-투자자 매칭 플랫폼 설계

> 작성일: 2026-01-29

## 1. 프로젝트 개요

AI 발달로 증가하는 1인 개발자(창업자)와 투자자를 연결하는 플랫폼.

### 핵심 특징
- **관계**: 개발자 1명 : 투자자 N명
- **플랫폼 역할**: 매칭/중개만 (계약은 당사자 간 개인 계약)
- **품질 관리**: 관리자가 프로젝트 심사 + 주간 보고 의무화
- **수익 모델**: 개발자 포인트 환전 시 수수료 (10%)

---

## 2. 액터

| 액터 | 설명 |
|-----|-----|
| 개발자 | 1인 창업자, 프로젝트 등록 및 투자 유치 |
| 투자자 | 프로젝트에 투자하는 사람 |
| 관리자 | 플랫폼 운영자, 프로젝트 심사 및 품질 관리 |

---

## 3. 유스케이스

### 3.1 개발자

**인증/가입**
- 회원가입 (OAuth)
- 로그인/로그아웃

**프로젝트 등록**
- 프로젝트 계획서 작성 (수익모델, 목표 투자금액, 최소 투자금액, 기술스택 등)
- 계획서 제출 → 심사 대기

**프로젝트 관리**
- 심사 결과 확인 (승인/반려)
- 반려 시 계획서 수정 후 재제출
- 주간 진행상황 보고서 제출
- 투자 모집 현황 조회 (현재 예치 금액, 참여 투자자 수)

**투자 진행**
- 목표액 달성 알림 수신
- 유튜브 라이브 진행 (투자자들과 소통)
- MVP 제출
- MVP 검토 결과 확인 (통과/실패)
- 최종 투자금액 확인 후 진행 여부 결정 (최소금액 미달 시)
- 주급 수령 (본 투자 시작 후)

**종료**
- 모집 종료 알림 수신 (미달성 시)
- 프로젝트 완료 처리

### 3.2 투자자

**인증/가입**
- 회원가입 (OAuth)
- 로그인/로그아웃

**포인트 관리**
- 현금 → 포인트 충전 (1:1)
- 포인트 → 현금 환전 (1:1, 수수료 없음)
- 포인트 잔액 조회

**프로젝트 탐색**
- 프로젝트 리스트 조회
- 프로젝트 검색/필터 (분야, 기술스택, 목표금액 등)
- 추천 프로젝트 조회
- 프로젝트 상세 조회 (계획서, 진행상황 보고서)

**투자 의향**
- 프로젝트에 포인트 예치 (투자 의향 표시)
- 예치 금액 수정/취소
- 내 투자 의향 목록 조회

**투자 진행**
- 목표액 달성 알림 수신
- 유튜브 라이브 참여
- MVP 결과물 확인
- 추가 투자 / 엑시트 결정
- 주간 보고서 열람 (본 투자 시작 후)

**종료**
- 모집 종료 알림 수신 (미달성 시)
- 미지급 포인트 반환 (개발자 중도 포기 시)

### 3.3 관리자

**프로젝트 심사**
- 심사 대기 프로젝트 목록 조회
- 계획서 검토 (수익모델, 실현가능성, 법적 제약)
- 프로젝트 승인/반려 (반려 시 사유 작성)

**품질 관리**
- 주간 보고서 미제출 개발자 확인
- 경고/제재 조치 (보고서 미제출 시)
- 프로젝트 강제 종료 (문제 발생 시)

**MVP 검증**
- MVP 제출물 검토
- MVP 통과/실패 판정

**회원 관리**
- 개발자/투자자 목록 조회
- 문제 회원 제재/탈퇴 처리

## 4. 투자 프로세스 흐름

```
1. [프로젝트 등록] 개발자가 계획서 작성 (목표금액 + 최소금액 설정)
        ↓
2. [심사] 관리자가 계획서 검토 → 승인/반려
        ↓
3. [모집] 투자자들이 포인트 예치 (투자 의향)
        ↓
   ┌────┴────┐
   ↓         ↓
 목표액     기간만료
 달성       (미달성)
   ↓         ↓
   │       [종료] 양쪽 알림
   ↓
4. [유튜브 라이브] 개발자-투자자 소통
        ↓
5. [MVP 개발] 1개월, 무급
        ↓
6. [MVP 검증] 관리자가 결과물 검토
        ↓
   ┌────┴────┐
   ↓         ↓
 통과       실패
   ↓         ↓
   │       [종료]
   ↓
7. [투자자 결정] 추가투자 or 엑시트
        ↓
8. [최종금액 확정]
        ↓
   ┌────┴────┐
   ↓         ↓
 ≥최소금액  <최소금액
   ↓         ↓
 자동진행   [개발자 선택]
   ↓         ↓
   │    ┌────┴────┐
   │    ↓         ↓
   │  진행함    포기함
   │    ↓         ↓
   └────┴────→    │
        ↓         ↓
9. [본투자 진행]  [종료]
   주간보고 + 주급
        ↓
10. [완료]
```

## 5. 프로젝트 상태

| 상태 | 설명 |
|-----|-----|
| `DRAFT` | 계획서 작성 중 |
| `PENDING_REVIEW` | 심사 대기 |
| `UNDER_REVIEW` | 심사 중 |
| `REJECTED` | 반려 (수정 후 재제출 가능) |
| `FUNDING` | 모집 중 |
| `MVP_IN_PROGRESS` | MVP 개발 중 (1개월) |
| `MVP_REVIEW` | MVP 검증 중 |
| `INVESTOR_DECISION` | 투자자 결정 대기 |
| `DEVELOPER_DECISION` | 개발자 진행 여부 결정 대기 |
| `ACTIVE` | 본투자 진행 중 |
| `COMPLETED` | 완료 |
| `CLOSED` | 종료 (사유는 별도 필드) |

**종료 사유 (CLOSED일 때):**
- `FUNDING_FAILED` - 모집 실패
- `MVP_FAILED` - MVP 검증 실패
- `DEVELOPER_QUIT` - 개발자 포기
- `DEVELOPER_REJECTED` - 개발자가 최소금액 미달로 진행 거부
- `ADMIN_CLOSED` - 관리자 강제 종료

## 6. 포인트 시스템

### 6.1 구조

```
[투자자]                    [플랫폼]                    [개발자]
   │                          │                          │
   │──(현금)──→ 포인트 충전    │                          │
   │           (1:1)          │                          │
   │                          │                          │
   │──(포인트)──→ 프로젝트 투자 ──(주급)──→ 포인트 수령   │
   │                          │                          │
   │←──(포인트)── 미지급분 반환 │  (개발자 중도포기 시)    │
   │                          │                          │
   │──(포인트)──→ 현금 환전    │                          │
   │           (1:1, 무료)    │                          │
   │                          │                          │
   │                          │   ←──(포인트)── 현금 환전 │
   │                          │              (10% 수수료) │
```

### 6.2 규칙

**투자자:**
- 현금 → 포인트: 1:1, 수수료 없음
- 포인트 → 현금: 1:1, 수수료 없음

**개발자:**
- 포인트 → 현금: 10% 수수료

**주급 지급:**
- 투자 확정 시 자동 송금 예약
- 개발 기간 동안 매주 포인트 지급

**중도 포기:**
- 미지급 주급 → 투자자 포인트로 반환
- 계약 분쟁은 당사자 간 해결 (플랫폼 무관)

### 6.3 플랫폼 수익
- 개발자 환전 수수료 (10%)

## 7. 법적 검토 필요 사항

### 7.1 메인 조사 항목

1. 온투법/자본시장법상 크라우드펀딩 규제 대상 여부
2. 전금법상 PG/에스크로 등록 요건
3. 유사수신행위 해당 여부
4. 등록 요건 (자본금, 인력 등)
5. 국내 크라우드펀딩 플랫폼 사례 분석
6. 리스크 회피 구조 (PG 에스크로 API, 통신판매중개)
7. 금융규제 샌드박스 사례
8. 투자 vs 후원 법적 구분
9. 주급 지급 시 근로계약 관계 오인 여부
10. 면책조항 유효 범위

### 7.2 포인트 시스템 관련

1. 선불전자지급수단 해당 여부
2. 게임머니/플랫폼 포인트 유사 사례
3. 규제 회피 가능한 구조
4. 플랫폼 면책 가능 범위
